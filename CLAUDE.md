# Claude Code プロジェクト設定

## PRレビュー

「PR#XXをレビューして」と依頼された場合：

1. PRの差分を確認し、コードレビューを実施する
2. レビュー内容をPRコメントとして投稿する
3. 不具合がなければマージする（軽微な改善提案はコメントに残すがマージはする）
4. 軽微でも不具合があればマージしない
5. 「マージしないでください」と明示的に言われた場合はマージしない

## Git操作

- コミットやプッシュの前に、対象ブランチに関連するPRがマージ済みでないか `gh pr list` 等で確認する
- マージ済みPRのブランチにプッシュしない。修正が必要な場合は新しいブランチ・新しいPRを作成する
