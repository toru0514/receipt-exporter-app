# Receipt Exporter App 運用計画

## フェーズ概要

| フェーズ | 期間目安 | 目的 |
|---------|---------|------|
| Phase 1 | 1-2週間 | 本番運用の基盤整備 |
| Phase 2 | 2-3週間 | 信頼性・品質の向上 |
| Phase 3 | 2-4週間 | 機能拡張・運用最適化 |

---

## Phase 1: 本番運用の基盤整備

### 1.1 環境構築・デプロイ

- [ ] Google Cloud Console でプロジェクト設定
  - OAuth 同意画面の構成（本番用リダイレクト URI 設定）
  - Gmail API, Sheets API, Generative AI API の有効化
  - OAuth クライアント ID の発行（本番用）
- [ ] デプロイ先の選定・設定
  - Vercel（推奨: Next.js との親和性が高い）
  - 環境変数の設定（`.env.local` → Vercel の環境変数管理）
  - カスタムドメインの設定（任意）
- [ ] HTTPS の確保（Vercel ならデフォルトで対応）

### 1.2 セキュリティ対策

- [ ] `NEXTAUTH_SECRET` の安全な生成と管理
- [ ] API エンドポイントへのレート制限の実装
- [ ] 入力値のバリデーション追加（メール HTML サイズ上限など）
- [ ] Content Security Policy (CSP) ヘッダーの設定
- [ ] 環境変数の漏洩防止確認（クライアント側に露出しないこと）

### 1.3 テスト基盤の導入

- [ ] テストフレームワーク導入（Vitest 推奨）
- [ ] ユニットテスト作成
  - `gemini.ts` — AI レスポンスのパース処理
  - `gmail.ts` — メール本文抽出ロジック
  - `sheets.ts` — スプレッドシート書き込みロジック
- [ ] API ルートの結合テスト作成
- [ ] CI パイプラインで自動テスト実行（GitHub Actions）

---

## Phase 2: 信頼性・品質の向上

### 2.1 エラーハンドリング強化

- [ ] API 呼び出しのリトライロジック実装（指数バックオフ）
- [ ] Gemini API レスポンスの JSON パース改善（正規表現 → 構造化出力）
- [ ] ユーザー向けエラーメッセージの改善
- [ ] 部分的な失敗時のリカバリ（解析失敗したメールのスキップ + リトライ UI）

### 2.2 パフォーマンス改善

- [ ] メール詳細取得の並列化（`Promise.all` または `Promise.allSettled`）
- [ ] メール取得のページネーション対応
- [ ] 解析結果のキャッシュ（同じメールの再解析を防止）
- [ ] ローディング UI の改善（スケルトンローダー）

### 2.3 データ品質の向上

- [ ] AI 抽出データのバリデーション（日付形式、金額の数値チェック等）
- [ ] 抽出結果の手動修正 UI（ユーザーが値を確認・修正してからエクスポート）
- [ ] 消費税の計算ロジック改善（注文単位 vs アイテム単位の整合性）

### 2.4 監視・ログ

- [ ] 構造化ログの導入（pino 等）
- [ ] エラートラッキングサービス導入（Sentry 推奨）
- [ ] API レスポンスタイム・成功率の計測
- [ ] Gemini API の使用量モニタリング（コスト管理）

## Phase 3: 機能拡張・運用最適化

### 3.1 機能追加

- [ ] メールの日付範囲指定フィルタ
- [ ] 検索キーワードフィルタ（商品名、金額など）

### 3.2 UX 改善

- [ ] ダークモード対応
- [ ] モバイルレスポンシブの最適化
- [ ] プログレスバー（解析進捗の可視化）
- [ ] 結果のソート・フィルタ機能

### 3.3 運用効率化

- [ ] README.md のプロジェクト固有の内容への更新
- [ ] Docker コンテナ化（デプロイ先の選択肢拡大）
- [ ] GitHub Actions による CI/CD パイプライン構築

- [ ] 定期実行機能（Cron / Scheduled Job によるメール自動取得・解析）

## インフラ構成（推奨）

```
[ユーザー]
    ↓ HTTPS
[Vercel / Next.js App]
    ├── Google OAuth (認証)
    ├── Gmail API (メール取得)
    ├── Gemini API (AI 解析)
    └── Google Sheets API (エクスポート)

[監視]
    ├── Sentry (エラートラッキング)
    ├── Vercel Analytics (パフォーマンス)
    └── Google Cloud Console (API 使用量)
```

## コスト見積もり

| サービス | 無料枠 | 超過時の想定コスト |
|---------|--------|-----------------|
| Vercel (Hobby) | 月100GBバンド幅 | 個人利用なら無料枠内 |
| Gemini API (Flash) | 日1,500リクエスト | 超過分は従量課金 |
| Gmail API | 日250クォータユニット/ユーザー | 通常無料 |
| Sheets API | 日300リクエスト/分 | 通常無料 |
| Sentry | 月5,000イベント | 無料枠で十分 |

**個人利用であれば、全体的にほぼ無料で運用可能。**

## 優先度サマリー

### 最優先（運用開始前に必須）
1. Google Cloud Console の本番設定
2. デプロイ環境の構築（Vercel）
3. `NEXTAUTH_SECRET` の安全な生成
4. 基本的なレート制限

### 高優先（運用開始後 早期に対応）
5. テスト基盤の導入
6. エラートラッキング（Sentry）
7. リトライロジック
8. Gemini パース処理の改善

### 中優先（安定運用後に対応）
9. パフォーマンス改善（並列処理）
10. データバリデーション
11. CI/CD パイプライン
12. 機能追加（フィルタ、CSV 等）
