# 作業者実施内容 - Phase 1: 本番運用の基盤整備

本ドキュメントは、Receipt Exporter App を本番環境で稼働させるために、手動で実施が必要な作業をステップバイステップで記載したものです。

---

## 1. Google Cloud Console の設定

### 1.1 プロジェクト作成

1. [Google Cloud Console](https://console.cloud.google.com/) にアクセスし、Google アカウントでログインする
2. 画面上部のプロジェクトセレクタ（「プロジェクトを選択」）をクリック
3. 「新しいプロジェクト」をクリック
4. 以下を入力して「作成」をクリック
   - **プロジェクト名**: `receipt-exporter-app`（任意の名前）
   - **場所**: 「組織なし」のまま
5. 作成完了後、プロジェクトセレクタで作成したプロジェクトを選択し、アクティブにする

> スクリーンショットポイント: プロジェクト作成完了後のダッシュボード画面（プロジェクト ID が表示されている状態）

### 1.2 API の有効化

以下の 3 つの API を有効化する。

1. 左メニューの「API とサービス」>「ライブラリ」を開く
2. **Gmail API** を検索し、クリックして「有効にする」を押す
3. **Google Sheets API** を検索し、クリックして「有効にする」を押す
4. **Generative Language API**（Gemini API）を検索し、クリックして「有効にする」を押す

> スクリーンショットポイント: 「API とサービス」>「有効な API とサービス」画面で、3 つの API が有効になっていることを確認

### 1.3 OAuth 同意画面の構成

1. 左メニューの「API とサービス」>「OAuth 同意画面」を開く
2. 「始める」もしくは「同意画面を構成」をクリック
3. 以下を設定する
   - **アプリ名**: `Receipt Exporter`
   - **ユーザーサポートメール**: 自分のメールアドレス
   - **デベロッパーの連絡先メールアドレス**: 自分のメールアドレス
4. 「保存して次へ」をクリック
5. **スコープ**の画面で「スコープを追加または削除」をクリックし、以下を追加する
   - `https://www.googleapis.com/auth/gmail.readonly`
   - `https://www.googleapis.com/auth/spreadsheets`
   - `https://www.googleapis.com/auth/generative-language`
   - `openid`
   - `email`
   - `profile`
6. 「更新」>「保存して次へ」をクリック
7. **テストユーザー**の画面で、自分の Gmail アドレスを追加する（公開ステータスが「テスト」の間は、ここに登録したユーザーのみ利用可能）
8. 「保存して次へ」で完了

> スクリーンショットポイント: OAuth 同意画面の設定完了後のサマリー画面

**注意**: 個人利用であれば「テスト」ステータスのままで問題ない。外部ユーザーに公開する場合は「本番環境に公開」を押し、Google の審査を受ける必要がある。

### 1.4 OAuth クライアント ID の発行

1. 左メニューの「API とサービス」>「認証情報」を開く
2. 「認証情報を作成」>「OAuth クライアント ID」をクリック
3. 以下を設定する
   - **アプリケーションの種類**: 「ウェブ アプリケーション」
   - **名前**: `Receipt Exporter Web Client`（任意）
   - **承認済みの JavaScript 生成元**:
     - `https://<your-vercel-domain>.vercel.app`（Vercel デプロイ後に判明する URL）
     - `http://localhost:3000`（ローカル開発用、必要に応じて）
   - **承認済みのリダイレクト URI**:
     - `https://<your-vercel-domain>.vercel.app/api/auth/callback/google`
     - `http://localhost:3000/api/auth/callback/google`（ローカル開発用、必要に応じて）
4. 「作成」をクリック
5. 表示される **クライアント ID** と **クライアント シークレット** を安全な場所にメモする

> スクリーンショットポイント: OAuth クライアント ID 作成完了後のダイアログ（クライアント ID とシークレットが表示されている画面）

**重要**: クライアントシークレットは一度しか表示されない場合がある。必ずこの時点でコピーして保存すること。

### 1.5 Gemini API キーの発行（任意）

OAuth トークンを使わず API キーで Gemini を利用する場合:

1. [Google AI Studio](https://aistudio.google.com/apikey) にアクセス
2. 「API キーを作成」をクリック
3. 先ほど作成したプロジェクトを選択
4. 生成された API キーを安全な場所にメモする

> この API キーは環境変数 `GOOGLE_GEMINI_API_KEY` として設定する。未設定の場合は OAuth トークンが使用される。

## 2. Vercel デプロイ設定

### 2.1 前提条件

- GitHub アカウントを持っていること
- 本プロジェクトのリポジトリが GitHub にプッシュ済みであること

### 2.2 Vercel アカウント作成・GitHub 連携

1. [Vercel](https://vercel.com/) にアクセス
2. 「Sign Up」をクリック
3. 「Continue with GitHub」を選択して GitHub アカウントで認証する
4. Vercel が GitHub へのアクセス許可を求めるので「Authorize」する
5. プランは **Hobby（無料）** を選択

> スクリーンショットポイント: Vercel ダッシュボードにログインした状態

### 2.3 プロジェクトのインポート

1. Vercel ダッシュボードで「Add New...」>「Project」をクリック
2. 「Import Git Repository」で対象のリポジトリ（`receipt-exporter-app`）を選択
   - リポジトリが表示されない場合は「Adjust GitHub App Permissions」からアクセス権を追加する
3. インポート設定画面で以下を確認
   - **Framework Preset**: `Next.js`（自動検出されるはず）
   - **Root Directory**: `.`（デフォルトのまま）
   - **Build Command**: `next build`（デフォルトのまま）
   - **Output Directory**: `.next`（デフォルトのまま）
4. **この時点ではまだ「Deploy」を押さない**（先に環境変数を設定する）

### 2.4 環境変数の設定

インポート設定画面の「Environment Variables」セクション、または「Settings」>「Environment Variables」で以下を設定する。

| 変数名 | 値 | 説明 |
|--------|-----|------|
| `GOOGLE_CLIENT_ID` | 手順 1.4 で取得したクライアント ID | Google OAuth クライアント ID |
| `GOOGLE_CLIENT_SECRET` | 手順 1.4 で取得したクライアントシークレット | Google OAuth クライアントシークレット |
| `NEXTAUTH_URL` | `https://<your-vercel-domain>.vercel.app` | デプロイ先の URL |
| `NEXTAUTH_SECRET` | 手順 3 で生成した値 | NextAuth セッション暗号化キー |
| `GOOGLE_GEMINI_API_KEY` | 手順 1.5 で取得した API キー（任意） | Gemini API キー（未設定なら OAuth トークン使用） |

**設定手順**:

1. 「Key」欄に変数名を入力
2. 「Value」欄に値を入力
3. **Environment** は「Production」「Preview」「Development」すべてにチェックを入れる
4. 「Add」をクリック
5. 全変数を追加し終えたら「Deploy」をクリック

> スクリーンショットポイント: 環境変数一覧画面（値は伏せ字で表示される）

### 2.5 デプロイの確認

1. デプロイが完了するまで待つ（通常 1-2 分）
2. デプロイ完了後、Vercel が割り当てた URL（`https://<project-name>.vercel.app`）にアクセス
3. アプリが正常に表示されることを確認

### 2.6 リダイレクト URI の更新

Vercel の URL が確定したら、Google Cloud Console 側の設定を更新する。

1. Google Cloud Console >「API とサービス」>「認証情報」を開く
2. 手順 1.4 で作成した OAuth クライアント ID をクリック
3. **承認済みの JavaScript 生成元** と **承認済みのリダイレクト URI** の `<your-vercel-domain>` 部分を、実際の Vercel ドメインに書き換える
4. 「保存」をクリック

> 反映まで数分かかる場合がある。

### 2.7 カスタムドメインの設定（任意）

独自ドメインを使用する場合:

1. Vercel ダッシュボードでプロジェクトを選択
2. 「Settings」>「Domains」を開く
3. 「Add Domain」にドメイン名を入力し「Add」をクリック
4. 表示される DNS レコード（CNAME または A レコード）を、ドメインの DNS 管理画面で設定する
5. DNS 反映後、Vercel が自動的に SSL 証明書を発行する

> カスタムドメインを設定した場合は、`NEXTAUTH_URL` 環境変数、Google Cloud Console のリダイレクト URI もそのドメインに更新すること。

### 2.8 HTTPS の確認

- Vercel はデフォルトで HTTPS を提供するため、特別な設定は不要
- ブラウザのアドレスバーに鍵マークが表示されていることを確認する
- カスタムドメインの場合も Vercel が Let's Encrypt の証明書を自動発行する

## 3. NEXTAUTH_SECRET の安全な生成

### 3.1 シークレットの生成

ターミナルで以下のコマンドを実行する:

```bash
openssl rand -base64 32
```

出力例:
```
aBcDeFgHiJkLmNoPqRsTuVwXyZ1234567890+/==
```

この出力値を `NEXTAUTH_SECRET` として使用する。

**注意事項**:
- 生成した値はパスワードマネージャー等で安全に保管すること
- 他のプロジェクトと同じシークレットを使い回さないこと
- Git リポジトリに直接コミットしないこと

### 3.2 Vercel 環境変数への登録

1. Vercel ダッシュボードでプロジェクトを選択
2. 「Settings」>「Environment Variables」を開く
3. 以下を入力
   - **Key**: `NEXTAUTH_SECRET`
   - **Value**: 手順 3.1 で生成した値
   - **Environment**: Production, Preview, Development すべてにチェック
4. 「Save」をクリック
5. 変更を反映するため、「Deployments」タブから最新のデプロイを「Redeploy」する
